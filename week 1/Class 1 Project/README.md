# Week 1 — Class 1 Project: Local LLM + Text Sorting Automation

## Achievement
- Run a LLM model locally via Ollama.
- Automate a text-sorting task using Python with production-grade structure.

---

## Production-Grade File Structure

```
week1_class1_project/
├── README.md                 # You are here
├── requirements.txt          # Pinned Python dependencies
├── .env.example              # Template for environment variables
├── config/
│   └── settings.yaml         # Human-readable configuration (strategy, categories, paths)
├── src/
│   ├── __init__.py           # Package marker
│   ├── llm_client.py         # Encapsulates all Ollama REST API calls
│   ├── text_sorter.py        # Core sorting logic (alphabetical, length, LLM category)
│   └── utils/
│       ├── __init__.py
│       └── file_handler.py   # Reusable file I/O utilities
├── data/
│   ├── raw/                  # Input text files
│   └── sorted/               # Output files generated by the sorter
└── main.py                   # Single entry point
```

## Step-by-Step Instructions

### Step 1: Install Ollama (Local LLM Engine)

**macOS / Linux:**
```bash
curl -fsSL https://ollama.com/install.sh | sh
```

**Windows:**
Download from [https://ollama.com/download](https://ollama.com/download)

**Verify installation:**
```bash
ollama --version
```

**ANALOGY:** Ollama is the *engine*. Right now you have installed the engine in your garage, but it is not running yet.

---

### Step 2: Pull a Model

```bash
ollama pull llama3.2
```

**WHY llama3.2?** It is a lightweight (~2GB) model from Meta, capable enough for categorization tasks and runnable on CPUs with 8GB+ RAM.

**Verify the model:**
```bash
ollama list
```

---

### Step 3: Start the Ollama Server

```bash
ollama serve
```

Leave this terminal open. It will listen on `http://localhost:11434`.

**ANALOGY:** This is turning the key and starting the engine. Until you do this, the car will not move.

---

### Step 4: Set Up Python Environment

**Option A: venv (Recommended for deployment literacy)**
```bash
cd week1_class1_project
python -m venv .venv
source .venv/bin/activate        # macOS/Linux
# OR
.venv\Scripts\activate         # Windows
pip install -r requirements.txt
```

**Option B: Conda (Recommended if you have GPU/CUDA needs)**
```bash
conda create -n week1class1 python=3.11
conda activate week1class1
pip install -r requirements.txt
```

---

### Step 5: Configure Environment Variables

```bash
cp .env.example .env
```

Edit `.env` if your Ollama runs on a different host (e.g., a remote server or different port).

**WHY .env?** Keeps secrets and machine-specific settings out of Git. Your teammate can have a different `.env` without merge conflicts.

---

### Step 6: Run the Project

```bash
python main.py
```

**What happens:**
1. `main.py` loads `config/settings.yaml`.
2. It checks for raw data. If none exists, it creates `data/raw/sample_texts.txt` with demo lines.
3. It initializes `TextSorter` with the strategy defined in config (default: `category`).
4. It sends each line to the local LLM via `OllamaClient`.
5. The LLM categorizes each line (technology, sports, politics, entertainment, other).
6. Results are written to `data/sorted/category_*.txt`.

---

### Step 7: Experiment with Strategies

Edit `config/settings.yaml`:

```yaml
sorting:
  strategy: "alphabetical"   # Change from "category" to "alphabetical" or "length"
```

Run again:
```bash
python main.py
```

**WHY?** To see how the same codebase handles classical algorithms vs. AI-powered logic with zero code changes — only config changes.

## Troubleshooting

| Issue | Solution |
|-------|----------|
| `Cannot connect to Ollama` | Ensure `ollama serve` is running in another terminal. |
| `Model not found` | Run `ollama pull llama3.2` first. |
| `ModuleNotFoundError` | Ensure your virtual environment is activated. |
| `Empty output files` | Check that `data/raw/*.txt` has non-empty lines. |
